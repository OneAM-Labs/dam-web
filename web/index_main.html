Platform Architecture

Designed with precise, water-infrastructure terminology for clear mental models.

Reservoir

A Reservoir encapsulates your entire project's continuity, storing configuration, streams, seals, and environment metadata in one place.

Collection

Files are intentionally gathered into the collection area based on strict purities rules, ready to be preserved in your next seal.

Seal

An immutable snapshot of your collected files in time. Once sealed, a state represents a permanent, restorable point in your project's continuity.

Stream

Parallel flows of development. You intentionally Flow into an isolated stream to experiment, without disturbing your main line of work.

.purities & .impurities

Deterministic configuration tracking. .purities acts as a strict allowlist. .impurities acts as a blocklist. No more accidental collects.

Timeline

The chronological evolution of your Reservoir. Inspect the full history of seals on your current stream at any time.

Sync

Push and pull your seal history to and from a linked cloud platform. GitHub is supported today, with more platforms planned.

Provider

A project-type profile — Flutter, Custom, and more to come — defining environment checks and setup commands for full project exports.

The Continuity Workflow

A predictable, deterministic approach to preserving state.

# Initialize a new Reservoir in your project $ dam source
# Collect files strictly governed by your .purities allowlists $ dam collect .
# Preserve the collected state as an immutable snapshot $ dam seal "Implemented core authentication flow"
# Safely flow into a new stream for isolated development $ dam flowinto feature-database
# Push your seal history to GitHub $ dam sync --action push

Installation

DAM is currently provided as standalone binaries for local-first execution.

Linux (x64) .zip 

curl -LO https://dam-pcp.web.app/downloads/dam-linux.zip
unzip dam-linux.zip
chmod +x dam
sudo mv dam /usr/local/bin/
Download Archive

Windows (x64) .zip 

curl -LO https://dam-pcp.web.app/downloads/dam-windows.zip
mkdir C:\dam
tar -xf dam-windows.zip -C C:\dam

setx PATH "%PATH%;C:\dam"

:: Restart your terminal to apply PATH changes
curl -LO https://dam-pcp.web.app/downloads/dam-windows.zip

New-Item -ItemType Directory -Force -Path C:\dam

Expand-Archive .\dam-windows.zip -DestinationPath C:\dam -Force

$path = [Environment]::GetEnvironmentVariable("Path", "User")
if ($path -notlike "*C:\dam*") {
    [Environment]::SetEnvironmentVariable(
        "Path",
        "$path;C:\dam",
        "User"
    )
}
